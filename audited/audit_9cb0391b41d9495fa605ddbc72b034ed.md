# Audit Report

## Title
Vault Status Bypass in Adaptor Value Update Functions Allows Unauthorized Manipulation of Operation Value Records

## Summary
Five public adaptor value update functions (`update_receipt_value()`, `update_navi_position_value()`, `update_cetus_position_value()`, `update_suilend_position_value()`, `update_momentum_position_value()`) lack vault status validation, allowing any user to manipulate asset values during vault operations. This enables attackers to bypass critical loss tolerance checks by updating asset values between operation Phase 2 (asset return) and Phase 3 (loss validation), directly compromising the protocol's loss tolerance enforcement mechanism.

## Finding Description

The vulnerability exists across all five adaptor value update functions, which are public and callable by any address without vault status checks.

**Receipt Adaptor Vulnerability:**
The `update_receipt_value()` function only validates the receipt vault's status but never checks the main vault's status [1](#0-0) 

**Other Adaptor Vulnerabilities:**
Similarly, the other four adaptors are public without ANY vault status checks:
- Navi adaptor [2](#0-1) 
- Cetus adaptor [3](#0-2) 
- Suilend adaptor [4](#0-3) 
- Momentum adaptor [5](#0-4) 

**Vault Operation Three-Phase Flow:**

During vault operations, there's a critical three-phase flow where assets are borrowed and must be properly accounted for:

**Phase 1 - Start Operation:** The vault status is set to `VAULT_DURING_OPERATION_STATUS` and borrowed assets are tracked [6](#0-5) 

When DeFi assets are borrowed (including receipts), they're added to the `asset_types_borrowed` tracking list [7](#0-6) 

**Phase 2 - End Operation:** Assets are returned and value update is enabled [8](#0-7) 

The `enable_op_value_update()` function sets `value_update_enabled = true` [9](#0-8) 

**Phase 3 - Value Update Check:** All borrowed assets must be updated and loss tolerance is enforced [10](#0-9) 

**The Vulnerability Window:**
Between Phase 2 and Phase 3, when the vault is in `VAULT_DURING_OPERATION_STATUS` with `value_update_enabled = true`, the `finish_update_asset_value()` function will mark assets as updated in the operation record [11](#0-10) 

Since all five adaptor update functions are public and call `finish_update_asset_value()`, any attacker can update asset values during this critical window, manipulating the `op_value_update_record` that's used for loss calculation.

**Loss Tolerance Impact:**
The loss tolerance check directly uses the total USD value calculated from these potentially manipulated asset values [12](#0-11) 

The total USD value aggregates all asset values from the `assets_value` table [13](#0-12) 

## Impact Explanation

**Critical Security Invariant Violation - Loss Tolerance Bypass:**
The protocol enforces a maximum loss tolerance per epoch to protect depositors. By inflating asset values during the operation window, an attacker can reduce the calculated loss in Phase 3, allowing operations that should fail the `ERR_EXCEED_LOSS_LIMIT` check to succeed. This directly violates the core security guarantee that losses are bounded per epoch.

**Operational Control Loss:**
Operators lose exclusive control over when and how asset values are updated during operations. Attackers can front-run operator transactions with manipulated oracle states, different timing, or altered receipt vault conditions, causing:
- Unpredictable total USD value calculations
- Operation failures from deflated values
- Loss of deterministic operation outcomes

**Accounting Integrity Corruption:**
The manipulated total USD value affects the share ratio calculation used for all subsequent deposits and withdrawals. This creates cascading accounting errors:
- Incorrect share pricing for new deposits
- Wrong withdrawal amounts for users
- Cumulative value drift across multiple operations

**Affected Stakeholders:**
- All vault depositors: Share values become unreliable
- Protocol operators: Operations become unpredictable
- Protocol treasury: Loss tolerance violations can drain reserves

## Likelihood Explanation

**Public Entry Points:**
All five adaptor update functions are explicitly declared as `public fun`, making them callable by any address on the blockchain. No capability checks or authorization mechanisms exist.

**Minimal Preconditions:**
- Vault must hold a DeFi asset (Receipt, Navi position, Cetus position, Suilend obligation, or Momentum position) - common in production
- Receipt vault (if applicable) must be in normal status - typical operational state
- Main vault must be between Phase 2 and Phase 3 - predictable window based on operator transaction patterns
- Oracle config and clock are public shared objects - universally accessible

**Practical Exploitation:**
1. Monitor vault `OperationEnded` events to detect Phase 2 completion
2. Immediately submit transaction calling appropriate `update_X_value()` function
3. Use current oracle prices (no manipulation needed) or wait for favorable price movement
4. Front-run operator's Phase 3 transaction

No complex transaction coordination, no capital requirements, no special privileges needed. Single transaction execution with ~0.01 SUI gas cost.

**Economic Viability:**
- **Attack Cost:** Minimal gas fees only
- **Potential Gain:** Bypass loss limits to enable continued malicious operations, or grief competitor vaults
- **Detection Risk:** Low - legitimate value updates are indistinguishable from malicious ones in transaction logs
- **Repeatability:** Can be executed every operation cycle

## Recommendation

**Immediate Fix:**
Add vault status check to all five adaptor update functions. The functions should only be callable when the vault is in `VAULT_NORMAL_STATUS`:

```move
public fun update_receipt_value<PrincipalCoinType, PrincipalCoinTypeB>(
    vault: &mut Vault<PrincipalCoinType>,
    receipt_vault: &Vault<PrincipalCoinTypeB>,
    config: &OracleConfig,
    clock: &Clock,
    asset_type: String,
) {
    vault.assert_normal();  // Add this check
    receipt_vault.assert_normal();
    // ... rest of function
}
```

Apply the same fix to:
- `update_navi_position_value()`
- `update_cetus_position_value()`
- `update_suilend_position_value()`
- `update_momentum_position_value()`

**Alternative Solution:**
Make these functions callable only by operators by adding operator capability checks, though this reduces flexibility for legitimate external value updates.

**Long-term Enhancement:**
Consider implementing a value update queue or commit-reveal scheme for operation periods to prevent front-running while maintaining price accuracy.

public fun end_op_with_bag<T, CoinType, ObligationType>(
    vault: &mut Vault<T>,
    operation: &Operation,
    cap: &OperatorCap,
    mut defi_assets: Bag,
    tx: TxBag,
    principal_balance: Balance<T>,
    coin_type_asset_balance: Balance<CoinType>,
) {
    vault::assert_operator_not_freezed(operation, cap);
    vault.assert_during_operation();

    let TxBag {
        vault_id,
        defi_asset_ids,
        defi_asset_types,
    } = tx;

    assert!(vault.vault_id() == vault_id, ERR_VAULT_ID_MISMATCH);

    let length = defi_asset_ids.length();
    let mut i = 0;
    while (i < length) {
        let defi_asset_id = defi_asset_ids[i];
        let defi_asset_type = defi_asset_types[i];

        if (defi_asset_type == type_name::get<NaviAccountCap>()) {
            let navi_asset_type = vault_utils::parse_key<NaviAccountCap>(defi_asset_id);
            let navi_account_cap = defi_assets.remove<String, NaviAccountCap>(navi_asset_type);
            vault.return_defi_asset(navi_asset_type, navi_account_cap);
        };

        if (defi_asset_type == type_name::get<CetusPosition>()) {
            let cetus_asset_type = vault_utils::parse_key<CetusPosition>(defi_asset_id);
            let cetus_position = defi_assets.remove<String, CetusPosition>(cetus_asset_type);
            vault.return_defi_asset(cetus_asset_type, cetus_position);
        };

        if (defi_asset_type == type_name::get<SuilendObligationOwnerCap<ObligationType>>()) {
            let suilend_asset_type = vault_utils::parse_key<
                SuilendObligationOwnerCap<ObligationType>,
            >(
                defi_asset_id,
            );
            let obligation = defi_assets.remove<String, SuilendObligationOwnerCap<ObligationType>>(
                suilend_asset_type,
            );
            vault.return_defi_asset(suilend_asset_type, obligation);
        };

        if (defi_asset_type == type_name::get<MomentumPosition>()) {
            let momentum_asset_type = vault_utils::parse_key<MomentumPosition>(defi_asset_id);
            let momentum_position = defi_assets.remove<String, MomentumPosition>(
                momentum_asset_type,
            );
            vault.return_defi_asset(momentum_asset_type, momentum_position);
        };

        if (defi_asset_type == type_name::get<Receipt>()) {
            let receipt_asset_type = vault_utils::parse_key<Receipt>(defi_asset_id);
            let receipt = defi_assets.remove<String, Receipt>(receipt_asset_type);
            vault.return_defi_asset(receipt_asset_type, receipt);
        };

        i = i + 1;
    };

    emit(OperationEnded {
        vault_id: vault.vault_id(),
        defi_asset_ids,
        defi_asset_types,
        principal_coin_type: type_name::get<T>(),
        principal_amount: principal_balance.value(),
        coin_type_asset_type: type_name::get<CoinType>(),
        coin_type_asset_amount: coin_type_asset_balance.value(),
    });

    vault.return_free_principal(principal_balance);

    if (coin_type_asset_balance.value() > 0) {
        vault.return_coin_type_asset<T, CoinType>(coin_type_asset_balance);
    } else {
        coin_type_asset_balance.destroy_zero();
    };

    vault.enable_op_value_update();

    defi_assets.destroy_empty();
}

public fun end_op_value_update_with_bag<T, ObligationType>(
    vault: &mut Vault<T>,
    operation: &Operation,
    cap: &OperatorCap,
    clock: &Clock,
    tx: TxBagForCheckValueUpdate,
) {
    vault::assert_operator_not_freezed(operation, cap);
    vault.assert_during_operation();

    let TxBagForCheckValueUpdate {
        vault_id,
        defi_asset_ids,
        defi_asset_types,
        total_usd_value,
        total_shares,
    } = tx;

    assert!(vault.vault_id() == vault_id, ERR_VAULT_ID_MISMATCH);

    // First check if all assets has been returned
    let length = defi_asset_ids.length();
    let mut i = 0;
    while (i < length) {
        let defi_asset_id = defi_asset_ids[i];
        let defi_asset_type = defi_asset_types[i];

        if (defi_asset_type == type_name::get<NaviAccountCap>()) {
            let navi_asset_type = vault_utils::parse_key<NaviAccountCap>(defi_asset_id);
            assert!(vault.contains_asset_type(navi_asset_type), ERR_ASSETS_NOT_RETURNED);
        };

        if (defi_asset_type == type_name::get<CetusPosition>()) {
            let cetus_asset_type = vault_utils::parse_key<CetusPosition>(defi_asset_id);
            assert!(vault.contains_asset_type(cetus_asset_type), ERR_ASSETS_NOT_RETURNED);
        };

        if (defi_asset_type == type_name::get<SuilendObligationOwnerCap<ObligationType>>()) {
            let suilend_asset_type = vault_utils::parse_key<
                SuilendObligationOwnerCap<ObligationType>,
            >(
                defi_asset_id,
            );
            assert!(vault.contains_asset_type(suilend_asset_type), ERR_ASSETS_NOT_RETURNED);
        };

        if (defi_asset_type == type_name::get<MomentumPosition>()) {
            let momentum_asset_type = vault_utils::parse_key<MomentumPosition>(defi_asset_id);
            assert!(vault.contains_asset_type(momentum_asset_type), ERR_ASSETS_NOT_RETURNED);
        };

        i = i + 1;
    };

    let total_usd_value_before = total_usd_value;
    vault.check_op_value_update_record();
    let total_usd_value_after = vault.get_total_usd_value(
        clock,
    );

    // Update tolerance if there is a loss (there is a max loss limit each epoch)
    let mut loss = 0;
    if (total_usd_value_after < total_usd_value_before) {
        loss = total_usd_value_before - total_usd_value_after;
        vault.update_tolerance(loss);
    };

    assert!(vault.total_shares() == total_shares, ERR_VERIFY_SHARE);

    emit(OperationValueUpdateChecked {
        vault_id: vault.vault_id(),
        total_usd_value_before,
        total_usd_value_after,
        loss,
    });

    vault.set_status(VAULT_NORMAL_STATUS);
    vault.clear_op_value_update_record();
}

public(package) fun update_tolerance<T0>(self: &mut Vault<T0>, loss: u256) {

    // let loss_limit = usd_value_before * (self.loss_tolerance as u256) / (RATE_SCALING as u256);

    emit(LossToleranceUpdated {
        vault_id: self.vault_id(),
        current_loss: self.cur_epoch_loss,
        loss_limit: loss_limit,
    });
}

public(package) fun finish_update_asset_value<PrincipalCoinType>(
    self: &mut Vault<PrincipalCoinType>,
    asset_type: String,
    usd_value: u256,
    now: u64,
) {
    self.check_version();
    self.assert_enabled();

    let last_update_time = &mut self.assets_value_updated[asset_type];
    *last_update_time = now;

    let position_value = &mut self.assets_value[asset_type];
    *position_value = usd_value;

    if (
        self.status() == VAULT_DURING_OPERATION_STATUS 
        && self.op_value_update_record.value_update_enabled 
        && self.op_value_update_record.asset_types_borrowed.contains(&asset_type)
    ) {
        self.op_value_update_record.asset_types_updated.add(asset_type, true);
    };

    emit(AssetValueUpdated {
        vault_id: self.vault_id(),
        asset_type: asset_type,
        usd_value: usd_value,
        timestamp: now,
    });
}

public(package) fun get_total_usd_value<PrincipalCoinType>(
    self: &Vault<PrincipalCoinType>,
    clock: &Clock,
): u256 {
    self.check_version();
    self.assert_enabled();

    let now = clock.timestamp_ms();
    let mut total_usd_value = 0;

    self.asset_types.do_ref!(|asset_type| {
        let last_update_time = *self.assets_value_updated.borrow(*asset_type);
        assert!(now - last_update_time <= MAX_UPDATE_INTERVAL, ERR_USD_VALUE_NOT_UPDATED);

        let usd_value = *self.assets_value.borrow(*asset_type);
        total_usd_value = total_usd_value + usd_value;
    });

    emit(TotalUSDValueUpdated {
        vault_id: self.vault_id(),
        total_usd_value: total_usd_value,
        timestamp: now,
    });

    total_usd_value
}

public(package) fun borrow_defi_asset<PrincipalCoinType, AssetType: key + store>(
    self: &mut Vault<PrincipalCoinType>,
    asset_type: String,
): AssetType {
    self.check_version();
    self.assert_enabled();

    assert!(contains_asset_type(self, asset_type), ERR_ASSET_TYPE_NOT_FOUND);

    if (self.status() == VAULT_DURING_OPERATION_STATUS) {
        self.op_value_update_record.asset_types_borrowed.push_back(asset_type);
    };

    emit(DefiAssetBorrowed {
        vault_id: self.vault_id(),
        asset_type: asset_type,
    });

    self.assets.remove<String, AssetType>(asset_type)
}
