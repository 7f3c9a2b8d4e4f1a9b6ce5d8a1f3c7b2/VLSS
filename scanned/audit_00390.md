## Update on Car Info Handling in drife_app.move

The functions `update_car_info_value` and `add_car_category_info` currently do not validate the `value` parameter, which is of type `vector<u64>`. This oversight may result in the acceptance of malformed inputs. 

Utilizing a structured data type with defined fields for specific car-related information introduces a layer of clarity and ensures that the data conforms to an expected structure.

## Function Implementations

### update_car_info_value

```rust
public fun update_car_info_value(_cap: &AdminCap, astorage: &mut AppStorage,
    car_type: u8, value: vector<u64>) {
    assert!(table::contains(&astorage.car_types_info, car_type), ENotValidCarType);
    *table::borrow_mut(&mut astorage.car_types_info, car_type) = value;
}
```

### add_car_category_info

```rust
public fun add_car_category_info(_cap: &AdminCap, astorage: &mut AppStorage,
    car_type: u8, value: vector<u64>) {
    assert!(!table::contains(&astorage.car_types_info, car_type), ECarTypeAlreadyExist);
    table::add<u8, vector<u64>>(&mut astorage.car_types_info, car_type, value);
}
```

## Remediation

Instead of using `vector<u64>`, consider introducing a `CarInfo` structure to ensure data integrity and clarity.
