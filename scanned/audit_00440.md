# Incorrect Function Call

## Summary
No summary provided.

## Details
## SUI Vault Parameter Modification

The admin has the capability to modify the parameters of the staked SUI vault.

## Update Atomic Unstake Max Fee

The `update_atomic_unstake_max_fee` function updates the max fee taken during the atomic unstake process. The protocol should ensure that `min_fee <= max_fee`, but the implemented assert demands that the new max be equal to or greater than the current max.

### Remediation

Replace `max_atomic_unstake_fee` with `min_atomic_unstake_fee`.

```diff
staked_sui_vault_state.move DIFF
@@ -1536,7 +1517,7 @@ module lsd::staked_sui_vault_state {
public(friend) fun update_atomic_unstake_max_fee(
    new_max_fee: u64,
) {
    assert!(new_max_fee <= FIXED_ONE, EInvalidPercentage);
-   assert!(max_atomic_unstake_fee(staked_sui_vault_state) <= new_max_fee,
,→ EInvalidPercentage);
+   assert!(min_atomic_unstake_fee(staked_sui_vault_state) <= new_max_fee,
,→ EInvalidPercentage);
```

### Patch

Fixed in commit `9881b46` by calling the correct function.
