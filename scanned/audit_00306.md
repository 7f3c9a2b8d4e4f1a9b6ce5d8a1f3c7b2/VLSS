# Incorrect Whitelist Check

## Summary
No summary provided.

## Details
## Code Overview

The function `check_token_and_min_trade_amount` utilizes the return value of `find_token_and_min_trade_amount` when `require_check_token_white_list` is true. The `is_existed` variable indicates whether the token is present in the white list. If it is true, the function should verify the minimum trade amount. Conversely, if `is_existed` is false, meaning the token is absent from the white list, the function should immediately return false.

## Code Snippet

> _config.moverust_

```rust
public fun check_token_and_min_trade_amount<Coin>(config: &GlobalConfig, amount: u64): bool {
    if (!config.require_check_token_white_list) {
        true
    } else {
        let (is_existed, min_trade_amount) = find_token_and_min_trade_amount<Coin>(config);
        if (!is_existed && min_trade_amount <= amount) {
            true
        } else {
            false
        }
    }
}
```

## Remediation

Change the condition to `is_existed && min_trade_amount <= amount`.
