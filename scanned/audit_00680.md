# Validate Tortuga Address

## Summary
No summary provided.

## Details
## Stake Router Vulnerability

In `stake_router.move`, several initialization and setter methods take `tortuga: &signer` as a function parameter. The lack of checks against this parameter could lead to accidental vulnerabilities.

## Code Example

```move
public entry fun set_reward_commission(
    tortuga: &signer,
    value: u64
) acquires StakingStatus {
    let staking_status =
        borrow_global_mut<StakingStatus>(signer::address_of(tortuga));
    staking_status.reward_commission = value;
}
```

## Remediation

It is recommended to include at least a check against the hardcoded `@tortuga` to ensure that the addresses match.

```move
assert!(
    signer::address_of(tortuga) == @tortuga,
    error::unauthenticated(EPERMISSION_DENIED)
);
```
