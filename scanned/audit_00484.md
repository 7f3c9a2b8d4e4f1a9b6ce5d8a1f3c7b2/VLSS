## Code Optimization in logic.move

During the execution of `update_state` in `logic.move`, the time difference is calculated separately within the `calculate_compounded_interest` and `calculate_linear_interest` functions. To enhance code efficiency and minimize redundant computations, we should remove this redundancy by calculating the time difference once in `update_state` and passing it as a parameter to both interest calculation functions.

## Current Implementation

```rust
    // e.g. get the time difference of the last update -->

    let timestamp_difference = (current_timestamp - last_update_timestamp as u256) / 1000;
    [...]
}

    [...]

```

## Remediation

1. Calculate the time difference once at the beginning of `update_state` and store it in a variable.
2. Pass this time difference as a parameter to the following functions:
    - `calculate_compounded_interest`
    - `calculate_linear_interest`
