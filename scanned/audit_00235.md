## Overview
This document outlines necessary improvements to the current implementation of the `increase_validator_stake` function and the `split_n_sui` function within the Liquid Staking module.

### 1. Increase Validator Stake Function

Add a check that ensures `sui_amount > MIN_STAKE_AMOUNT` at the beginning of `increase_validator_stake`. This ensures that if the `sui_amount` passed to the function is less than or equal to `MIN_STAKE_AMOUNT`, the function exits early, returning zero. This prevents further processing if the amount does not meet the minimum requirement.

```rust
contracts/sources/liquid_staking.move
public fun increase_validator_stake<P>(
    self: &mut LiquidStakingInfo<P>,
    _: &AdminCap<P>,
    system_state: &mut SuiSystemState,
    validator_address: address,
    sui_amount: u64,
    ctx: &mut TxContext
): u64 {
    self.refresh(system_state, ctx);
    let sui = self.storage.split_up_to_n_sui_from_sui_pool(sui_amount);
    if (sui.value() < MIN_STAKE_AMOUNT) {
        self.storage.join_to_sui_pool(sui);
        return 0;
    };
    [...]
}
```

### 2. Split N SUI Function

In the current implementation of `split_n_sui`, there is no check to ensure that the function only attempts to split when the requested amount of SUI is actually available in the `sui_pool`. Before calling `split_from_sui_pool`, a check should be added to verify whether the `sui_pool.value` is at least equal to `max_sui_amount_out`.
