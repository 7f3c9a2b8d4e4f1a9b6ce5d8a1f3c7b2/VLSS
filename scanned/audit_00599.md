# Missing Flow Address Length Check

## Summary
No summary provided.

## Details
## Teleport Functionality Review

The function `teleport::teleport_to_flow` includes a parameter `flow_addr` that is checked for validity using the `teleport::is_flow_addr_valid` function. However, this check is missing from `teleport::teleport_from_flow`.

## Code Reference

```rust
sources/teleport.move
338 // check flow tx is valid
339 assert!(is_flow_tx_valid(flow_tx_hash), EFLOW_TX_INVALID);
```

While the transaction hash is checked for validity using the `is_flow_tx_valid` function, the length of the `flow_addr` parameter is not verified.

## Remediation

To ensure the validity of the `flow_addr` parameter, it is recommended to include a check for its length inside the `teleport_from_flow` function.

```rust
assert!(is_flow_addr_valid(flow_addr), EFLOW_ADDR_INVALID);
```
