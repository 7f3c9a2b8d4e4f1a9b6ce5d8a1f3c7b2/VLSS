## Changes Made

1. **Modify `force_transfer_amount`** 
   - The `ForceTransferred` event is now only emitted once, preventing duplicate emissions from the internal call to `force_transfer` and in `force_transfer_amount`.

2. **Update `add_holder_address_internal`**
   - Changed to utilize the `group_id` parameter instead of the hardcoded `DEFAULT_GROUP`. This enhances flexibility, allowing the function to add addresses to any specified group rather than being restricted to the default group.

   ```rust
   // Source: sources/rules/group_holder_config/group_holder_config.move
   fun add_holder_address_internal<T>(
   self: &mut Config,
   holder_id: u64,
   addr: address,
   group_id: u64,
   ) {
       event::emit(HolderAddressAdded<T> { holder_id: holder_id, addr });
       self.holders[holder_id].insert(addr);
       self.addresses.add(addr, AddressInfo { holder_id, group_id });
       self.groups[&default_group()].add_holder_address(holder_id, addr);
   }
   ```

3. **Streamline `shared_token`** 
   - Both `action_request` and `request_inner` perform the same function, so one of them has been removed for simplicity and clarity.

Implement the above-mentioned refactors.
