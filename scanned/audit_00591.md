## Details
## Module Testing Functions

The `init_module_test` functions in the **admin** and **fees** modules are intended for testing purposes only, as they initialize configurations to a given account. However, these functions have been left public and are not decorated with the `#[test_only]` function modifier, meaning they are accessible to the public.

## Code Snippets

### Admin Module

```rust
public fun init_module_test(account: &signer) {
    move_to(account, ConfigV2 {
        admin: @Streamflow,
        streamflow_fees: 25,
        treasury: @Streamflow,
        withdrawer: @Streamflow,
        tx_fee: DEFAULT_TX_FEE
    })
}
```

### Fees Module

```rust
public fun init_module_test(account: &signer) {
    move_to(account, FeeTable {
        values: table::new()
    })
}
```

## Remediation

Convert the public `init_module_test` functions in the admin and fees modules to test-only by adding the `#[test_only]` function modifier.
