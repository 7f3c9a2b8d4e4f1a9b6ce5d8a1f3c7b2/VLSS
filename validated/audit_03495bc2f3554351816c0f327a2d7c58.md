# Audit Report

## Title
Zero-Liquidity Momentum Position Causes Vault DoS Through Unnecessary Price Validation

## Summary
The `get_position_value()` function in the Momentum adaptor performs unconditional pool-oracle price validation even when positions have zero liquidity. When pool prices deviate beyond slippage tolerance, the function aborts, preventing value updates for empty positions and blocking vault operation completion, causing a complete protocol DoS.

## Finding Description

The root cause lies in the `get_position_value()` function which performs price validation regardless of position liquidity. The function first retrieves token amounts, then unconditionally validates that the pool price is within slippage tolerance of oracle prices before calculating the final value [1](#0-0) 

For zero-liquidity positions (amount_a = 0, amount_b = 0), this price validation is unnecessary since the position value would be zero anyway, yet the assertion can still abort the entire transaction.

When an operator borrows a Momentum position during an operation, the vault status is set to `VAULT_DURING_OPERATION_STATUS` [2](#0-1) 

All borrowed assets must have their values updated before the operation can complete. When `update_momentum_position_value` is called, if the price validation fails, the function aborts before reaching `finish_update_asset_value` [3](#0-2) 

This means the asset is never marked as updated in the operation value update record. When attempting to finalize the operation via `end_op_value_update_with_bag`, the `check_op_value_update_record` function validates that all borrowed assets were updated [4](#0-3) 

If any asset was not marked as updated, this check fails with `ERR_USD_VALUE_NOT_UPDATED`, preventing the vault from transitioning back to `VAULT_NORMAL_STATUS` [5](#0-4) 

**Security Guarantee Broken:** The vault's operation state machine can become permanently stuck, violating the guarantee that operations can always be completed and the vault can return to normal operational status.

**Critical Recovery Limitation:** Admin intervention via `set_vault_enabled` is blocked because it explicitly checks that the vault is not in `VAULT_DURING_OPERATION_STATUS` [6](#0-5) 

Similarly, operator removal of the problematic position via `remove_defi_asset_support` requires the vault to be in normal status [7](#0-6) 

## Impact Explanation

**Operational DoS Impact:**
- The vault becomes permanently stuck in `VAULT_DURING_OPERATION_STATUS`
- All user deposits and withdrawals are blocked because both `request_deposit` and `request_withdraw` require `VAULT_NORMAL_STATUS` via the `assert_normal()` check [8](#0-7) [9](#0-8) 
- The vault cannot process any new operations until the issue is resolved

**Affected Parties:**
- All vault users lose access to their deposited funds (cannot withdraw)
- New users cannot deposit
- The operator cannot complete the current operation or start new operations
- The entire vault becomes non-functional

**Severity Justification - Medium:**
1. Complete operational disruption affecting all vault functionality
2. No direct fund loss - user funds remain in the vault
3. Recovery requires waiting for market prices to realign within tolerance (time-dependent)
4. Admin and operator intervention mechanisms are blocked by status checks
5. Users' funds are safe but completely inaccessible during the DoS period

## Likelihood Explanation

**Entry Point:** Standard operator workflow calling `update_momentum_position_value` during required operation value updates.

**Preconditions (All Realistic):**
1. A Momentum position with zero liquidity exists in the vault - common after full liquidity withdrawal or if liquidity was never properly added
2. The Momentum pool's spot price deviates from oracle prices beyond the configured slippage tolerance - occurs naturally due to:
   - Market volatility causing rapid pool price movements
   - Oracle update delays vs real-time pool prices
   - MEV activity or high-frequency trading causing temporary price dislocations

**Execution Practicality:**
- No attacker actions required - natural market conditions trigger it
- Zero-liquidity positions are standard in AMM protocols
- Price deviations beyond slippage tolerance occur regularly in volatile markets
- No economic cost to trigger - just requires waiting for market conditions

**Affected Adaptors:**
The same vulnerability pattern exists in the Cetus adaptor with identical unconditional price validation logic [10](#0-9) 

**Probability Assessment:** Medium-to-High probability given the frequency of zero-liquidity positions and natural market price deviations in DeFi environments.

## Recommendation

Add an early return for zero-liquidity positions before performing price validation:

```move
public fun get_position_value<CoinA, CoinB>(
    pool: &MomentumPool<CoinA, CoinB>,
    position: &MomentumPosition,
    config: &OracleConfig,
    clock: &Clock,
): u256 {
    let (amount_a, amount_b, sqrt_price) = get_position_token_amounts(pool, position);
    
    // Early return for zero-liquidity positions - no need for price validation
    if (amount_a == 0 && amount_b == 0) {
        return 0
    };
    
    let type_name_a = into_string(get<CoinA>());
    let type_name_b = into_string(get<CoinB>());
    
    // ... rest of the function with price validation
}
```

Apply the same fix to the Cetus adaptor's `calculate_cetus_position_value` function.

## Notes

This vulnerability demonstrates a design flaw where safety checks (price validation) are applied too broadly without considering edge cases (zero-liquidity positions). The impact is amplified by the vault's state machine design which prevents admin/operator recovery when stuck in operation status. Recovery requires either waiting for natural price realignment or emergency contract upgrades.

```text
public fun get_position_value<CoinA, CoinB>(
    pool: &MomentumPool<CoinA, CoinB>,
    position: &MomentumPosition,
    config: &OracleConfig,
    clock: &Clock,
): u256 {
    let (amount_a, amount_b, sqrt_price) = get_position_token_amounts(pool, position);

    let type_name_a = into_string(get<CoinA>());
    let type_name_b = into_string(get<CoinB>());

    let decimals_a = config.coin_decimals(type_name_a);
    let decimals_b = config.coin_decimals(type_name_b);

    // Oracle price has 18 decimals
    let price_a = vault_oracle::get_asset_price(config, clock, type_name_a);
    let price_b = vault_oracle::get_asset_price(config, clock, type_name_b);
    let relative_price_from_oracle = price_a * DECIMAL / price_b;

    let pool_price = sqrt_price_x64_to_price(sqrt_price, decimals_a, decimals_b);
    let slippage = config.dex_slippage();
    assert!(
        (pool_price.diff(relative_price_from_oracle) * DECIMAL  / relative_price_from_oracle) < (DECIMAL  * slippage / SLIPPAGE_BASE),
        ERR_INVALID_POOL_PRICE,
    );

    let normalized_price_a = vault_oracle::get_normalized_asset_price(config, clock, type_name_a);
    let normalized_price_b = vault_oracle::get_normalized_asset_price(config, clock, type_name_b);

    let value_a = vault_utils::mul_with_oracle_price(amount_a as u256, normalized_price_a);
    let value_b = vault_utils::mul_with_oracle_price(amount_b as u256, normalized_price_b);

    value_a + value_b
}
```

```text
public fun end_op_value_update_with_bag<T, ObligationType>(
    vault: &mut Vault<T>,
    operation: &Operation,
    cap: &OperatorCap,
    clock: &Clock,
    tx: TxBagForCheckValueUpdate,
) {
    vault::assert_operator_not_freezed(operation, cap);
    vault.assert_during_operation();

    let TxBagForCheckValueUpdate {
        vault_id,
        defi_asset_ids,
        defi_asset_types,
        total_usd_value,
        total_shares,
    } = tx;

    assert!(vault.vault_id() == vault_id, ERR_VAULT_ID_MISMATCH);

    // First check if all assets has been returned
    let length = defi_asset_ids.length();
    let mut i = 0;
    while (i < length) {
        let defi_asset_id = defi_asset_ids[i];
        let defi_asset_type = defi_asset_types[i];

        if (defi_asset_type == type_name::get<NaviAccountCap>()) {
            let navi_asset_type = vault_utils::parse_key<NaviAccountCap>(defi_asset_id);
            assert!(vault.contains_asset_type(navi_asset_type), ERR_ASSETS_NOT_RETURNED);
        };

        if (defi_asset_type == type_name::get<CetusPosition>()) {
            let cetus_asset_type = vault_utils::parse_key<CetusPosition>(defi_asset_id);
            assert!(vault.contains_asset_type(cetus_asset_type), ERR_ASSETS_NOT_RETURNED);
        };

        if (defi_asset_type == type_name::get<SuilendObligationOwnerCap<ObligationType>>()) {
            let suilend_asset_type = vault_utils::parse_key<
                SuilendObligationOwnerCap<ObligationType>,
            >(
                defi_asset_id,
            );
            assert!(vault.contains_asset_type(suilend_asset_type), ERR_ASSETS_NOT_RETURNED);
        };

        if (defi_asset_type == type_name::get<MomentumPosition>()) {
            let momentum_asset_type = vault_utils::parse_key<MomentumPosition>(defi_asset_id);
            assert!(vault.contains_asset_type(momentum_asset_type), ERR_ASSETS_NOT_RETURNED);
        };

        i = i + 1;
    };

    let total_usd_value_before = total_usd_value;
    vault.check_op_value_update_record();
    let total_usd_value_after = vault.get_total_usd_value(
        clock,
    );

    // Update tolerance if there is a loss (there is a max loss limit each epoch)
    let mut loss = 0;
    if (total_usd_value_after < total_usd_value_before) {
        loss = total_usd_value_before - total_usd_value_after;
        vault.update_tolerance(loss);
    };

    assert!(vault.total_shares() == total_shares, ERR_VERIFY_SHARE);

    emit(OperationValueUpdateChecked {
        vault_id: vault.vault_id(),
        total_usd_value_before,
        total_usd_value_after,
        loss,
    });

    vault.set_status(VAULT_NORMAL_STATUS);
    vault.clear_op_value_update_record();
}
```

```text
public(package) fun request_deposit<PrincipalCoinType>(
    self: &mut Vault<PrincipalCoinType>,
    coin: Coin<PrincipalCoinType>,
    clock: &Clock,
    expected_shares: u256,
    receipt_id: address,
    recipient: address,
): u64 {
    self.check_version();
    self.assert_normal();
    assert!(self.contains_vault_receipt_info(receipt_id), ERR_RECEIPT_NOT_FOUND);

    let vault_receipt = &mut self.receipts[receipt_id];
    assert!(vault_receipt.status() == NORMAL_STATUS, ERR_WRONG_RECEIPT_STATUS);

    // Generate current request id
    let current_deposit_id = self.request_buffer.deposit_id_count;
    self.request_buffer.deposit_id_count = current_deposit_id + 1;

    // Deposit amount
    let amount = coin.value();

    // Generate the new deposit request and add it to the vault storage
    let new_request = deposit_request::new(
        current_deposit_id,
        receipt_id,
        recipient,
        self.id.to_address(),
        amount,
        expected_shares,
        clock.timestamp_ms(),
    );
    self.request_buffer.deposit_requests.add(current_deposit_id, new_request);

    emit(DepositRequested {
        request_id: current_deposit_id,
        receipt_id: receipt_id,
        recipient: recipient,
        vault_id: self.id.to_address(),
        amount: amount,
        expected_shares: expected_shares,
    });

    // Temporary buffer the coins from user
    // Operator will retrieve this coin and execute the deposit
    self.request_buffer.deposit_coin_buffer.add(current_deposit_id, coin);

    vault_receipt.update_after_request_deposit(amount);

    current_deposit_id
}
```

```text
        current_request_id,
        receipt_id,

        self.id.to_address(),
        shares,
        expected_amount,
        clock.timestamp_ms(),
    );
    self.request_buffer.withdraw_requests.add(current_request_id, new_request);

    emit(WithdrawRequested {
        request_id: current_request_id,
        receipt_id: receipt_id,
        recipient: recipient,
        vault_id: self.id.to_address(),
        shares: shares,
        expected_amount: expected_amount,
    });

    vault_receipt.update_after_request_withdraw(shares, recipient);
```

```text
public(package) fun remove_defi_asset_support<PrincipalCoinType, AssetType: key + store>(
    self: &mut Vault<PrincipalCoinType>,
    idx: u8,
): AssetType {
    self.check_version();
    self.assert_normal();

    let asset_type = vault_utils::parse_key<AssetType>(idx);

    let (contains, index) = self.asset_types.index_of(&asset_type);
    assert!(contains, ERR_ASSET_TYPE_NOT_FOUND);
    self.asset_types.remove(index);

    let asset_value = self.assets_value[asset_type];
    let asset_value_updated = self.assets_value_updated[asset_type];
    assert!(asset_value == 0 || asset_value_updated == 0, ERR_ASSET_TYPE_NOT_FOUND);

    emit(DefiAssetRemoved {
        vault_id: self.vault_id(),
        asset_type: asset_type,
    });

    self.assets.remove<String, AssetType>(asset_type)
}
```

```text
public fun calculate_cetus_position_value<CoinTypeA, CoinTypeB>(
    pool: &mut CetusPool<CoinTypeA, CoinTypeB>,
    position: &CetusPosition,
    config: &OracleConfig,
    clock: &Clock,
): u256 {
    let position_id = object::id(position);

    let (amount_a, amount_b) = pool.get_position_amounts(position_id);

    let type_name_a = into_string(get<CoinTypeA>());
    let type_name_b = into_string(get<CoinTypeB>());

    let decimals_a = config.coin_decimals(type_name_a);
    let decimals_b = config.coin_decimals(type_name_b);

    // Oracle price has 18 decimals
    let price_a = vault_oracle::get_asset_price(config, clock, type_name_a);
    let price_b = vault_oracle::get_asset_price(config, clock, type_name_b);
    let relative_price_from_oracle = price_a * DECIMAL / price_b;

    // e.g. For SUI-USDC Pool, decimal_a = 9, decimal_b = 6
    // pool price = 3e18
    // price_a = 3e18
    // price_b = 1e18
    // relative_price_from_oracle = 3e18 * 1e18 / 1e18 = 3e18

    // pool price = price_a / price_b (not consider decimals)
    let pool_price = sqrt_price_x64_to_price(pool.current_sqrt_price(), decimals_a, decimals_b);
    let slippage = config.dex_slippage();
    assert!(
        (pool_price.diff(relative_price_from_oracle) * DECIMAL  / relative_price_from_oracle) < (DECIMAL  * slippage / SLIPPAGE_BASE),
        ERR_INVALID_POOL_PRICE,
    );

    let normalized_price_a = vault_oracle::get_normalized_asset_price(config, clock, type_name_a);
    let normalized_price_b = vault_oracle::get_normalized_asset_price(config, clock, type_name_b);

    let value_a = vault_utils::mul_with_oracle_price(amount_a as u256, normalized_price_a);
    let value_b = vault_utils::mul_with_oracle_price(amount_b as u256, normalized_price_b);

    value_a + value_b
}
```
