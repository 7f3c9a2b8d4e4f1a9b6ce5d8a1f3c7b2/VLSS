# Audit Report

## Title
Permanent Vault DoS via Unimplemented Momentum Liquidity Math Functions

## Summary
The `liquidity_math` module contains only stub implementations that unconditionally abort with `abort 0`, causing any vault operation involving MomentumPosition assets to permanently lock the vault in DURING_OPERATION_STATUS with no admin recovery mechanism. All user funds become permanently inaccessible.

## Finding Description

The Volo vault protocol integrates with Momentum DEX positions through the momentum adaptor, but the required `liquidity_math` module contains only unimplemented stub functions. All functions in this module simply execute `abort 0` without any implementation. [1](#0-0) 

The momentum adaptor's `update_momentum_position_value()` function relies on `get_position_token_amounts()`, which calls the stub implementation `liquidity_math::get_amounts_for_liquidity()`. [2](#0-1) 

When an operator initiates a vault operation that borrows a MomentumPosition asset, the vault's status transitions to DURING_OPERATION_STATUS and the asset is recorded in `asset_types_borrowed`. [3](#0-2) [4](#0-3) 

After the operation returns all borrowed assets, the system enables value updates by calling `enable_op_value_update()`. [5](#0-4) 

To complete the operation and transition the vault back to NORMAL status, the operator must call `end_op_value_update_with_bag()`, which invokes `check_op_value_update_record()`. This function strictly enforces that ALL borrowed assets must have their values updated. [6](#0-5) [7](#0-6) 

Since updating MomentumPosition values requires calling the stub `liquidity_math` functions which always abort, the asset can never be marked as updated in `asset_types_updated`. This causes `check_op_value_update_record()` to always fail, preventing the operation from completing.

The admin's `set_vault_enabled()` function explicitly checks that the vault status is NOT DURING_OPERATION_STATUS before allowing any status changes. [8](#0-7) 

Additionally, operators cannot remove the problematic MomentumPosition asset once the vault is stuck, because `remove_defi_asset_support()` requires the vault to be in NORMAL status. [9](#0-8) 

## Impact Explanation

**Permanent Vault Lockup**: Once triggered, the vault becomes permanently stuck in DURING_OPERATION_STATUS. The only function that can change the status back to NORMAL is `end_op_value_update_with_bag()`, but it requires `check_op_value_update_record()` to pass first, which is impossible.

**Complete Fund Lockup**: All user funds held in the vault become permanently inaccessible. Users cannot deposit, withdraw, or execute any vault operations while the vault is in DURING_OPERATION_STATUS.

**No Recovery Mechanism**: There is no admin emergency function to bypass this lock. The admin cannot use `set_vault_enabled()` because it explicitly prevents status changes during operations. No other function in the codebase can modify vault status to recover from this state.

**Severity**: Critical - This results in permanent DoS with total fund lockup affecting all vault shareholders and depositors.

## Likelihood Explanation

**Entry Points**: Operators can add MomentumPosition assets to the vault using `add_new_defi_asset()` and borrow them in operations using `start_op_with_bag()`. [10](#0-9) [11](#0-10) 

**Feasibility**: This vulnerability requires only:
1. A vault with a MomentumPosition asset (added by operator)
2. An operator borrowing it in an operation (normal operation)
3. Attempting to update the position value (required to complete operation)

**Trigger Condition**: The vulnerability can occur accidentally if operators attempt to use the Momentum integration believing it is implemented, or intentionally by a malicious operator. The presence of stub implementations suggests this is incomplete code that should not be in production.

**Detection**: This is a deterministic vulnerability - if MomentumPosition assets are used, the vault WILL be permanently locked. It is not probabilistic.

**Overall Likelihood**: High for vaults that integrate Momentum positions, medium-to-high overall given the stub implementations indicate incomplete development.

## Recommendation

1. **Immediate**: Remove or disable the Momentum adaptor integration until `liquidity_math` functions are fully implemented.

2. **Short-term**: Implement all stub functions in `liquidity_math.move` with proper mathematical calculations for liquidity position valuation.

3. **Long-term**: Add an emergency admin recovery function that can reset vault status with appropriate safeguards and multi-sig requirements:

```move
public fun emergency_reset_operation_status<PrincipalCoinType>(
    _: &AdminCap,
    vault: &mut Vault<PrincipalCoinType>,
) {
    vault.check_version();
    // Clear operation records
    vault.clear_op_value_update_record();
    // Reset to normal status
    vault.set_status(VAULT_NORMAL_STATUS);
}
```

4. **Testing**: Add integration tests that verify all adaptor value update functions before allowing production deployment.

public fun get_position_value<CoinA, CoinB>(
    pool: &MomentumPool<CoinA, CoinB>,
    position: &MomentumPosition,
    config: &OracleConfig,
    clock: &Clock,
): u256 {
    let (amount_a, amount_b, sqrt_price) = get_position_token_amounts(pool, position);

    let type_name_a = into_string(get<CoinA>());
    let type_name_b = into_string(get<CoinB>());

    let decimals_a = config.coin_decimals(type_name_a);
    let decimals_b = config.coin_decimals(type_name_b);

    // Oracle price has 18 decimals
    let price_a = vault_oracle::get_asset_price(config, clock, type_name_a);
    let price_b = vault_oracle::get_asset_price(config, clock, type_name_b);
    let relative_price_from_oracle = price_a * DECIMAL / price_b;

    let pool_price = sqrt_price_x64_to_price(sqrt_price, decimals_a, decimals_b);
    let slippage = config.dex_slippage();
    assert!(
        (pool_price.diff(relative_price_from_oracle) * DECIMAL  / relative_price_from_oracle) < (DECIMAL  * slippage / SLIPPAGE_BASE),
        ERR_INVALID_POOL_PRICE,
    );

    let normalized_price_a = vault_oracle::get_normalized_asset_price(config, clock, type_name_a);
    let normalized_price_b = vault_oracle::get_normalized_asset_price(config, clock, type_name_b);

    let value_a = vault_utils::mul_with_oracle_price(amount_a as u256, normalized_price_a);
    let value_b = vault_utils::mul_with_oracle_price(amount_b as u256, normalized_price_b);

    value_a + value_b
}

public fun get_position_token_amounts<CoinA, CoinB>(
    pool: &MomentumPool<CoinA, CoinB>,
    position: &MomentumPosition,
): (u64, u64, u128) {
    let sqrt_price = pool.sqrt_price();

    let lower_tick = position.tick_lower_index();
    let upper_tick = position.tick_upper_index();

    let lower_tick_sqrt_price = tick_math::get_sqrt_price_at_tick(lower_tick);
    let upper_tick_sqrt_price = tick_math::get_sqrt_price_at_tick(upper_tick);

    let liquidity = position.liquidity();

    let (amount_a, amount_b) = liquidity_math::get_amounts_for_liquidity(
        sqrt_price,
        lower_tick_sqrt_price,
        upper_tick_sqrt_price,
        liquidity,
        false,
    );
    (amount_a, amount_b, sqrt_price)
}

    operation: &Operation,
    cap: &OperatorCap,
    clock: &Clock,
    defi_asset_ids: vector<u8>,
    defi_asset_types: vector<TypeName>,
    principal_amount: u64,
    coin_type_asset_amount: u64,
    ctx: &mut TxContext,
): (Bag, TxBag, TxBagForCheckValueUpdate, Balance<T>, Balance<CoinType>) {

    pre_vault_check(vault, ctx);

    let mut defi_assets = bag::new(ctx);

    let defi_assets_length = defi_asset_ids.length();
    assert!(defi_assets_length == defi_asset_types.length(), ERR_ASSETS_LENGTH_MISMATCH);

    let mut i = 0;
    while (i < defi_assets_length) {
        let defi_asset_id = defi_asset_ids[i];
        let defi_asset_type = defi_asset_types[i];

        if (defi_asset_type == type_name::get<NaviAccountCap>()) {
            let navi_asset_type = vault_utils::parse_key<NaviAccountCap>(defi_asset_id);
            let navi_account_cap = vault.borrow_defi_asset<T, NaviAccountCap>(
                vault_utils::parse_key<NaviAccountCap>(defi_asset_id),
            );
            defi_assets.add<String, NaviAccountCap>(navi_asset_type, navi_account_cap);
        };

            let cetus_asset_type = vault_utils::parse_key<CetusPosition>(defi_asset_id);

            defi_assets.add<String, CetusPosition>(cetus_asset_type, cetus_position);
        };

        if (defi_asset_type == type_name::get<SuilendObligationOwnerCap<ObligationType>>()) {
            let obligation_asset_type = vault_utils::parse_key<
                SuilendObligationOwnerCap<ObligationType>,
            >(
                defi_asset_id,
            );
            let obligation = vault.borrow_defi_asset<T, SuilendObligationOwnerCap<ObligationType>>(
                obligation_asset_type,
            );
            defi_assets.add<String, SuilendObligationOwnerCap<ObligationType>>(
                obligation_asset_type,
                obligation,
            );
        };

            let momentum_asset_type = vault_utils::parse_key<MomentumPosition>(defi_asset_id);

            let receipt = vault.borrow_defi_asset<T, Receipt>(receipt_asset_type);
            defi_assets.add<String, Receipt>(receipt_asset_type, receipt);
        };

        i = i + 1;
    };

    let principal_balance = if (principal_amount > 0) {
        vault.borrow_free_principal(principal_amount)
    } else {
        balance::zero<T>()
    };

    let coin_type_asset_balance = if (coin_type_asset_amount > 0) {
        vault.borrow_coin_type_asset<T, CoinType>(
            coin_type_asset_amount,
        )
    } else {
        balance::zero<CoinType>()
    };

    let total_usd_value = vault.get_total_usd_value(clock);
    let total_shares = vault.total_shares();

    let tx = TxBag {

        defi_asset_ids,
        defi_asset_types,
    };

        vault_id: vault.vault_id(),
        defi_asset_ids,
        defi_asset_types,
        total_usd_value,
        total_shares,
    };

        vault_id: vault.vault_id(),
        defi_asset_ids,
        defi_asset_types,
        principal_coin_type: type_name::get<T>(),
        principal_amount,
        coin_type_asset_type: type_name::get<CoinType>(),
        coin_type_asset_amount,
        total_usd_value,
    });

    (defi_assets, tx, tx_for_check_value_update, principal_balance, coin_type_asset_balance)
}

    defi_assets.destroy_empty();
}

        i = i + 1;
    };

    self: &mut Vault<PrincipalCoinType>,

    emit(DefiAssetRemoved {
        vault_id: self.vault_id(),
        asset_type: asset_type,
    });

    self.assets.remove<String, AssetType>(asset_type)

public(package) fun borrow_defi_asset<PrincipalCoinType, AssetType: key + store>(
    self: &mut Vault<PrincipalCoinType>,
    asset_type: String,
): AssetType {
    self.check_version();
    self.assert_enabled();

    assert!(contains_asset_type(self, asset_type), ERR_ASSET_TYPE_NOT_FOUND);

    if (self.status() == VAULT_DURING_OPERATION_STATUS) {
        self.op_value_update_record.asset_types_borrowed.push_back(asset_type);
    };

    emit(DefiAssetBorrowed {
        vault_id: self.vault_id(),
        asset_type: asset_type,
    });

    self.assets.remove<String, AssetType>(asset_type)
}
